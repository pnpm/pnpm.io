## 次要更改

### 目录的新设置

添加了 [`cleanupUnusedCatalogs`](/settings#cleanupunusedcatalogs) 配置。 当设置为 `true` 时，pnpm 将在安装过程中移除未使用的目录条目 [#9793](https://github.com/pnpm/pnpm/pull/9793)。

<!-- truncate -->

### 配置依赖关系改进

pnpm 现在将自动从 [配置依赖项](/config-dependencies) 加载名为 `@*/pnpm-plugin-*` 的 pnpmfiles [#9780](https://github.com/pnpm/pnpm/issues/9780)。

### config get/set 命令的变更

- `pnpm config get` 现在打印对象值的 INI 字符串 [#9797](https://github.com/pnpm/pnpm/issues/9797)。
- `pnpm config get` 现在接受属性路径。

  示例：

  ```sh
  pnpm config get catalog.react
  pnpm config get .catalog.react
  pnpm config get 'packageExtensions["@babel/parser"].peerDependencies["@babel/types"]'
  ```

  `pnpm config set` 现在接受以点开头或下标开头的键。

  示例：

  ```sh
  pnpm config set .ignoreScripts true
  ```
- `pnpm config get --json` 现在打印配置值的 JSON 序列化，而 `pnpm config set --json` 现在将输入值解析为 JSON。

## 补丁更改

- **半破坏性**。当自动安装缺少的对等依赖项时，优先选择根工作区包的直接依赖项中已经存在的版本 [#9835](https://github.com/pnpm/pnpm/pull/9835)。
- 执行 `pnpm create` 命令时，即使缓存已经存在，也必须验证 Node 版本是否受支持 [#9775](https://github.com/pnpm/pnpm/pull/9775)。
- 当对非缩写包发出请求时，请将 `*/*` 添加到 `Accept` 标头，以避免在 AWS CodeArtifact [#9862](https://github.com/pnpm/pnpm/issues/9862) 上收到 406 错误。
- pnpm 的独立 exe 版本再次与 glibc 2.26 兼容 [#9734](https://github.com/pnpm/pnpm/issues/9734)。
- 修复 `pnpm dlx pkg --help` 没有将 `--help` 传递给 `pkg` 的回归问题 [#9823](https://github.com/pnpm/pnpm/issues/9823)。
